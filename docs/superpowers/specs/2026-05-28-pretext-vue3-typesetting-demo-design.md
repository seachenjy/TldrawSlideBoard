# Pretext Vue3 图文排版 Demo 设计文档

## 1. 目标

基于 `Vue3` 和 `@chenglou/pretext` 构建一个图文排版 demo，用最小交互成本展示内容结构化表达和排版渲染能力。

本 demo 的核心目标：

- 展示 `pretext` 对标题、段落、图片、引用、代码块、分割线等基础内容块的表达能力
- 通过轻量工具栏体现内容插入和即时预览的编辑流程
- 采用简洁文档风，突出可读性和层级感
- 保持实现范围可控，不扩展为完整文档产品

## 2. 非目标

以下内容不在本次 demo 范围内：

- 拖拽排序
- 历史记录和撤销重做
- 登录、保存、云端同步
- 多文档管理
- 权限系统
- 复杂富文本工具栏
- 完整所见即所得编辑体验

## 3. 方案结论

采用“顶部工具栏 + 单页文档预览”的方案。

页面分为两层：

- 顶部为轻量工具栏，提供基础内容块插入能力
- 主区域为单页文档预览，直接展示当前 `pretext` 文档的排版效果

该方案相较于左右分栏编辑器更聚焦视觉结果，也更适合在空项目中快速搭建一个能稳定演示图文排版能力的 demo。

## 4. 页面结构

### 4.1 顶部工具栏

工具栏固定在页面顶部，承载最少但足够的插入动作。

建议按钮包括：

- 插入标题
- 插入段落
- 插入图片
- 插入引用
- 插入代码块
- 插入分割线

工具栏右侧可选增加一个轻量视图切换入口：

- 示例预览
- 结构视图

“结构视图”用于展示当前 `pretext` 文档的原始结构或序列化结果，用来强调该 demo 不只是视觉页面，而是基于结构化内容数据驱动。

### 4.2 文档预览区

主区域居中显示一篇单页文档，整体表现为简洁文档风：

- 页面背景使用浅色
- 文档主体使用白色卡片或浅边界容器
- 内容宽度控制在适合阅读的范围
- 标题、导语、正文、引用、代码块之间有明确间距
- 图片为横版内容图，服务于正文而不是抢占主体

进入页面时默认加载一份完整示例内容，避免用户先看到空白文档。

## 5. 组件拆分

### 5.1 `App`

负责页面壳子和布局组织：

- 挂载工具栏
- 挂载文档预览区
- 控制可选结构视图的显示状态

### 5.2 `EditorToolbar`

负责触发基础插入行为，不维护复杂业务状态。

职责：

- 渲染基础插入按钮
- 触发插入事件
- 在必要时展示轻量错误提示

### 5.3 `DocumentPreview`

负责接收当前文档数据并渲染成最终页面内容。

职责：

- 渲染默认示例文章
- 在插入新内容块后即时更新
- 在数据异常时显示降级错误卡片

### 5.4 `SourcePanel`

作为可选的辅助展示区域。

职责：

- 展示当前文档结构
- 只读显示序列化结果
- 帮助说明 `pretext` 的内容表达方式

该组件不参与编辑，不影响主链路。

### 5.5 `useDemoDocument`

用于管理 demo 文档状态和插入方法。

职责：

- 初始化默认示例文档
- 提供插入标题、段落、图片、引用、代码块、分割线的方法
- 对外暴露当前文档状态

所有插入逻辑统一放在这个组合式函数中，避免分散在多个组件内。

## 6. 数据流设计

数据流采用单向更新模式：

1. 页面初始化时，`useDemoDocument` 创建默认示例文档
2. `App` 将文档状态传递给 `DocumentPreview`
3. `EditorToolbar` 触发插入动作
4. 插入动作调用 `useDemoDocument` 中的对应方法
5. 文档状态更新后，`DocumentPreview` 自动重新渲染
6. 如果启用 `SourcePanel`，则从同一份状态只读展示结构数据

此方案避免引入全局状态管理，适合当前 demo 的轻量目标。

## 7. 示例内容设计

默认内容使用一篇简短但完整的图文文章，以便覆盖多种内容块。

建议内容结构：

- 主标题
- 导语段落
- 二级标题
- 正文段落
- 配图
- 引用块
- 代码块
- 分割线
- 总结段落

内容题材建议选择“产品设计与技术表达”或类似兼具文字与代码展示的主题，以便让引用块和代码块显得自然，而不是为了展示组件生硬拼接。

所有示例文案应使用真实可读内容，不使用“这里是一段正文”之类的占位文本。

## 8. 视觉设计

整体视觉方向为“简洁文档风”。

视觉原则：

- 控制色彩数量，避免过度装饰
- 强调留白和阅读节奏
- 标题层级清晰
- 图片与引用块适度强调
- 代码块有明显区分，但不抢占整页视觉重心

建议表现：

- 工具栏采用浅背景和细边框
- 预览文档区域采用中等宽度内容列
- 段落行高偏宽，优化长文阅读
- 引用块使用弱强调背景或左侧边线
- 代码块使用独立背景和等宽字体

## 9. 交互边界

本 demo 的交互聚焦于“插入内容并即时查看结果”。

交互规则：

- 默认有一份示例文章
- 点击工具栏按钮后，在文档末尾追加对应内容块
- 插入完成后主预览区立刻更新
- 结构视图只做辅助展示，不支持直接编辑

不扩展的交互：

- 不支持块级拖拽排序
- 不支持块级选中编辑面板
- 不支持文本级格式化工具
- 不支持保存与恢复

## 10. 错误处理

错误处理遵循“不打断演示”的原则。

- 插入失败时，在工具栏附近展示轻量错误提示
- 图片加载失败时，显示统一占位容器和说明文字，保证布局稳定
- 文档数据异常时，`DocumentPreview` 显示降级错误卡片，而不是整页白屏
- `SourcePanel` 解析失败时，只影响自身显示，不影响主预览区

## 11. 测试策略

### 11.1 手动验证

必须覆盖以下关键路径：

- 页面可以正常启动
- 默认示例内容可以完整渲染
- 每个工具栏按钮都可以插入对应内容块
- 插入后预览区即时更新
- 图片、引用块、代码块样式表现稳定

### 11.2 自动化测试

只添加少量高价值测试，避免为 demo 增加过多维护成本。

建议测试点：

- `useDemoDocument` 的插入方法能够正确追加对应 block
- 关键预览组件能够根据文档状态渲染基础内容

不要求：

- 完整端到端测试
- 大量视觉快照测试
- 复杂交互回归测试

## 12. 验收标准

当满足以下条件时，视为本 demo 完成：

- 基于 `Vue3` 和 `@chenglou/pretext` 成功运行
- 页面为“顶部工具栏 + 单页预览”结构
- 默认展示一篇完整图文文章
- 支持插入标题、段落、图片、引用、代码块、分割线
- 插入后能够即时反映在预览区
- 页面视觉符合简洁文档风
- 结构视图可选展示，且不影响主体验

## 13. 实现建议

实现时优先保证以下顺序：

1. 搭建基础 Vue3 项目和 `pretext` 接入
2. 先做默认文章渲染
3. 再实现工具栏插入动作
4. 最后补充结构视图和样式细节

该顺序可以保证尽快产出一个可见、可演示、可迭代的最小版本。
